# Product Hunt Research KB

Local markdown knowledge base for the one-day Codex product selection.

## Files

- `signals.md` - market signals and cited Product Hunt examples.
- `scoring.md` - selection criteria for a 2 x 5 hour Codex build.
- `candidates.md` - shortlisted product ideas with scores.
- `decision.md` - recommended direction and execution outline.

## Memory Principle

Keep the always-loaded context small and human-readable. Store durable facts,
scores, and decisions in markdown files; only pull detailed notes when needed.
This follows a file-based memory pattern: markdown is the shared source of truth
for humans and agents, while chat context stays lightweight.
